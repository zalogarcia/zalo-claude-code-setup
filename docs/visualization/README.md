# Interactive Architecture Visualization

Single-file interactive visualization of the whole Claude Code Pro Setup. Shows the orchestrator, agents, rules, commands, hooks, MCP servers, and how a request flows through the system.

**Live:** https://zalogarcia.github.io/zalo-claude-code-setup/visualization/

## Files

- `index.html` — the app. Opens from `file://` with no server once data is inlined (see Regenerating below).
- `data.json` — graph nodes + edges, generated by the scanner.
- `flows.json` — request-flow scenarios (6 canonical + 17 auto-generated), generated by the scanner.
- `generate-data.mjs` — scanner. Walks the repo, emits `data.json` + `flows.json`, optionally inlines into `index.html`.
- `preview/` — static preview images used by the repo README.

## Regenerating after repo changes

```bash
# Regenerate data + flows only
node docs/visualization/generate-data.mjs

# Regenerate AND inline into index.html (needed for file:// to work without a server)
node docs/visualization/generate-data.mjs --inline
```

Acceptance checks fail the script with exit code 1 if the tree looks unexpectedly thin (≥ 7 agents, ≥ 11 rules, ≥ 22 commands, ≥ 6 hooks, 8 MCP, ≥ 4 skills, all edges reference real nodes).

## Local dev

The HTML loads graph data from an inlined `<script type="application/json">` block first. If that's empty, it falls back to `fetch('data.json')` — which requires an HTTP server, because browsers block `file://` fetches for CORS reasons. Two ways to dev:

```bash
# Option A: use a local server (any static server)
python3 -m http.server 8731
# then open http://localhost:8731/docs/visualization/

# Option B: inline the data and open directly
node docs/visualization/generate-data.mjs --inline
open docs/visualization/index.html
```

## Tech

- Vanilla JS + SVG. No framework, no build pipeline.
- D3 v7 from jsdelivr (force simulation + zoom/pan only).
- Dark mode primary, light mode via `prefers-color-scheme` + header toggle.
- Inter (UI) + JetBrains Mono (markers/code) from Google Fonts.

## Deployment (GitHub Pages)

Pages is configured to serve from `/docs` on `main`. The live URL is the one linked above. No CI step is required — a regenerated + inlined `index.html` committed to `main` goes live automatically.
